# md2docx 产品迭代计划（2026 Q2）

> 版本：v1.0  
> 制定日期：2026-03-08  
> 适用阶段：2026-03-09 至 2026-06-30

## 1. 目标与原则

- 目标 1：把当前 MVP 提升为“可团队稳定使用”的 Beta 版本。
- 目标 2：优先保障转换准确性与可观测性，再扩展高级格式能力。
- 目标 3：形成可运维的部署方案（Docker + GCP）并降低环境依赖风险。

迭代原则：
- 先稳定后扩展：先做正确率和可追溯，再做高级样式。
- 小步快跑：每两周一个里程碑，保证可发布可回滚。
- 数据驱动：以转换成功率、歧义率、人工修订时长作为关键指标。

## 2. 里程碑规划

## M1（2026-03-09 ~ 2026-03-22）：稳定性强化

范围：
- 标题映射规则增强：编号、别名、大小写、空白与符号鲁棒性。
- 报告增强：输出章节级耗时、歧义候选原因、错误码分层（解析/映射/渲染/写出）。
- Web API 错误语义统一（HTTP 状态码 + 机器可读 detail）。

验收标准：
- 标题匹配成功率 >= 95%（在内部样例集上）。
- 歧义章节可定位率 100%（报告能给出候选路径与索引）。

## M2（2026-03-23 ~ 2026-04-12）：格式能力扩展

范围：
- 表格映射（Markdown table -> Word table）。
- 代码块样式模板化（字体、背景、边框可配置）。
- Mermaid 渲染缓存（按 hash 复用）与失败重试策略。

验收标准：
- 表格转换可用率 >= 90%（常见 3 种表格结构）。
- Mermaid 失败率较当前降低 50%。

## M3（2026-04-13 ~ 2026-05-10）：模板治理与批量能力

范围：
- 模板配置中心（每模板独立匹配策略文件）。
- 批量转换（目录输入，批量输出与汇总报告）。
- 增加“预检模式”（只校验不输出文档）。

验收标准：
- 批量 50 篇文档连续运行成功率 >= 98%。
- 预检能提前发现 >90% 的模板不匹配问题。

## M4（2026-05-11 ~ 2026-06-30）：可运维与发布

范围：
- 部署能力完善：健康检查、日志切分、基础监控指标。
- 安全与治理：上传大小限制、文件类型白名单、请求速率限制。
- 发布 Beta：版本化发布说明、回滚文档、演示材料。

验收标准：
- 单实例稳定运行 7 天无崩溃。
- 发布版本具备“可部署、可监控、可回滚”三项能力。

## 3. 功能优先级（Backlog）

P0（必须）：
- 标题匹配稳定性
- 报告可观测性
- Web/CLI 行为一致性
- Docker/GCP 部署可复用

P1（应做）：
- 表格支持
- 批量转换
- 预检模式

P2（可选）：
- 前端历史记录
- 用户鉴权
- 多模板管理 UI

## 4. 指标与度量

- 转换成功率：`成功文档数 / 总文档数`
- 标题歧义率：`ambiguous 章节数 / 总章节数`
- Mermaid 失败率：`mermaid_failures / mermaid 块总数`
- 人工修订时长：单文档平均修订分钟数

目标（2026-06-30）：
- 转换成功率 >= 98%
- 标题歧义率 <= 3%
- Mermaid 失败率 <= 2%
- 人工修订时长下降 50%

## 5. 风险与应对

- 风险：不同模板样式差异大导致映射不稳定。  
  应对：模板配置中心 + 预检模式。
- 风险：Mermaid 在不同环境渲染不一致。  
  应对：容器固定版本 + 渲染缓存 + 降级策略。
- 风险：批量处理时性能波动。  
  应对：分批执行、并发上限、失败重试与断点续跑。

- 完成 M1 的错误码分层与报告字段扩展。
- 增加 `--dry-run`（预检）命令雏形。
- 在 GCP 测试环境完成一次完整部署演练。
